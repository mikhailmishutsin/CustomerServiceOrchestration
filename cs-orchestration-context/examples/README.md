# Examples

All payload examples are sanitized. Do not replace masked values with real customer data.

## OMS examples

- `oms-search-orders-response.json` — current default sample for the order-search operation (`GET /search`) with `expand=true`.
- `oms-search-orders-response-expanded-sanitized.json` — same expanded search response with details and tracking status.
- `oms-get-order-details-response-not-expanded-sanitized.json` — details response without embedded tracking status.
- `oms-get-order-details-response-expanded-sanitized.json` — details response with embedded tracking status.

## Helpdesk examples

- `helpdesk-ticket-webhook.json` — sample normalized helpdesk webhook.
- `helpdesk-update-payload.json` — sample common helpdesk update payload.
